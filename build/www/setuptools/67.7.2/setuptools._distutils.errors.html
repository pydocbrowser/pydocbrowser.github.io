setuptools._distutils.errors

module documentation
(source)

distutils.errors Provides exceptions used by the Distutils modules. Note that Distutils modules may raise standard exceptions; in particular, SystemExit is usually raised for errors that are obviously the end-user's fault (eg. bad command-line arguments). This module is safe to use in "from ... import *" mode; it only exports symbols whose names start with "Distutils" and end with "Error".

Exception CCompilerError Some compile/link operation failed.
Exception CompileError Failure to compile one or more C/C++ source files.
Exception DistutilsArgError Raised by fancy_getopt in response to getopt.error -- ie. an error in the command line usage.
Exception DistutilsByteCompileError Byte compile error.
Exception DistutilsClassError Some command class (or possibly distribution class, if anyone feels a need to subclass Distribution) is found not to be holding up its end of the bargain, ie. implementing some part of the "command "interface.
Exception DistutilsError The root of all Distutils evil.
Exception DistutilsExecError Any problems executing an external program (such as the C compiler, when compiling C files).
Exception DistutilsFileError Any problems in the filesystem: expected file not found, etc. Typically this is for problems that we detect before OSError could be raised.
Exception DistutilsGetoptError The option table provided to 'fancy_getopt()' is bogus.
Exception DistutilsInternalError Internal inconsistencies or impossibilities (obviously, this should never be seen if the code is working!).
Exception DistutilsModuleError Unable to load an expected module, or to find an expected class within some module (in particular, command modules and classes).
Exception DistutilsOptionError Syntactic/semantic errors in command options, such as use of mutually conflicting options, or inconsistent options, badly-spelled values, etc. No distinction is made between option values originating in the setup script, the command line, config files, or what-have-you -- but if we *know* something originated in the setup script, we'll raise DistutilsSetupError instead.
Exception DistutilsPlatformError We don't know how to do something on the current platform (but we do know how to do it on some platform) -- eg. trying to compile C files on a platform not supported by a CCompiler subclass.
Exception DistutilsSetupError For errors that can be definitely blamed on the setup script, such as invalid keyword arguments to 'setup()'.
Exception DistutilsTemplateError Syntax error in a file list template.
Exception LibError Failure to create a static library from one or more C/C++ object files.
Exception LinkError Failure to link one or more C/C++ object files into an executable or shared library file.
Exception PreprocessError Failure to preprocess one or more C/C++ files.
Exception UnknownFileError Attempt to process an unknown file type.
