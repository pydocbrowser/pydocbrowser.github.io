class Win32EventsTestsBuilder(ReactorBuilder): (source)

View In Hierarchy

Builder defining tests relating to IReactorWin32Events.

Method test_addEvent When an event which has been added to the reactor is set, the action associated with the event is invoked in the reactor thread.
Method test_disconnectedOnError If the event handler raises an exception, the event is removed from the reactor and the handler's connectionLost method is called in the I/O thread and the exception is logged.
Method test_disconnectOnReturnValue If the event handler returns a value, the event is removed from the reactor and the handler's connectionLost method is called in the I/O thread.
Method test_interface An instance of the reactor has all of the methods defined on IReactorWin32Events.
Method test_ioThreadDoesNotChange Using IReactorWin32Events.addEvent does not change which thread is reported as the I/O thread.
Method test_notDisconnectedOnShutdown Event handlers added with IReactorWin32Events.addEvent do not have connectionLost called on them if they are still active when the reactor shuts down.
