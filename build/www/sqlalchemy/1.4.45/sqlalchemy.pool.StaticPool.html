sqlalchemy.pool.StaticPool

class documentation

class StaticPool(Pool): (source)

View In Hierarchy

A Pool of exactly one connection, used for all requests. Reconnect-related functions such as ``recycle`` and connection invalidation (which is also used to support auto-reconnect) are only partially supported right now and may not yield good results.

overrides sqlalchemy.pool.Pool.recreate

Return a new :class:`_pool.Pool`, of the same class as this one and configured with identical creation arguments. This method is used in conjunction with :meth:`dispose` to close out an entire :class:`_pool.Pool` and create a new one in its place.
