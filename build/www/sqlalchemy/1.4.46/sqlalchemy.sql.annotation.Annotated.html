sqlalchemy.sql.annotation.Annotated

class documentation

class Annotated(object): (source)

Known subclasses: sqlalchemy.sql.elements.AnnotatedColumnElement, sqlalchemy.sql.selectable.AnnotatedFromClause

View In Hierarchy

clones a SupportsAnnotated and applies an 'annotations' dictionary. Unlike regular clones, this clone also mimics __hash__() and __cmp__() of the original element so that it takes its place in hashed collections. A reference to the original element is maintained, for the important reason of keeping its hash value current. When GC'ed, the hash value may be reused, causing conflicts. .. note:: The rationale for Annotated producing a brand new class, rather than placing the functionality directly within ClauseElement, is **performance**. The __hash__() method is absent on plain ClauseElement which leads to significantly reduced function call overhead, as the use of sets and dictionaries against ClauseElement objects is prevalent, but most are not "annotated".
